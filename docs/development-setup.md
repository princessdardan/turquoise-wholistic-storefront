## Prerequisites

| Tool | Version | Check |
|---|---|---|
| Node.js | >= 20 | `node --version` |
| Yarn | 4.12.0 (via Corepack) | `yarn --version` |
| PostgreSQL | 16 | Required by the Medusa backend |

**Enable Yarn 4 via Corepack** (run once on the machine):

```bash
corepack enable
corepack prepare yarn@4.12.0 --activate
```

**Medusa v2 backend** must be running at `http://localhost:9000`. The backend repository lives at `../turquoise-wholistic` (sibling directory). See the backend `README.md` for its own setup steps.

## Installation

```bash
# 1. Clone the repository
git clone <repo-url> turquoise-wholistic-storefront
cd turquoise-wholistic-storefront

# 2. Install dependencies (Yarn 4 reads packageManager field in package.json)
yarn install

# 3. Create the local environment file
cp .env.template .env.local   # if a template exists, otherwise create manually
```

Then fill in the required environment variables as described in the next section.

## Environment Variables

Create a `.env.local` file in the project root. The only variable that is strictly required to start the application is `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY` — the build will exit with an error if it is missing (enforced by `check-env-variables.js`).

| Variable | Required | Default | Description |
|---|---|---|---|
| `MEDUSA_BACKEND_URL` | Yes | `http://localhost:9000` | URL of the Medusa v2 backend. Used by server-side data fetching. |
| `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY` | **Yes** | — | Fallback publishable API key. Build fails without it. Obtain from Medusa admin under Settings → API Keys. |
| `NEXT_PUBLIC_MEDUSA_RETAIL_PUBLISHABLE_KEY` | No | — | Retail sales channel publishable key. Generated by `setup-sales-channels.ts` in the backend. Required to show the channel selection splash. |
| `NEXT_PUBLIC_MEDUSA_PROFESSIONAL_PUBLISHABLE_KEY` | No | — | Professional sales channel publishable key. Generated by the same setup script. Required to show the channel selection splash. |
| `NEXT_PUBLIC_BASE_URL` | Yes | `http://localhost:8000` | Public URL of the storefront. Used for absolute link generation. |
| `NEXT_PUBLIC_DEFAULT_REGION` | No | `us` | Fallback region ISO-2 code when middleware cannot detect a region from the URL. |
| `NEXT_PUBLIC_STRIPE_KEY` | No | — | Stripe publishable key. Enables Stripe payment in checkout when present. |
| `NEXT_PUBLIC_TURNSTILE_SITE_KEY` | No | — | Cloudflare Turnstile site key. Contact and other public forms degrade gracefully when absent. |
| `TURNSTILE_SECRET_KEY` | No | — | Cloudflare Turnstile server-side secret. Required for server-side CAPTCHA verification. |
| `REVALIDATE_SECRET` | No | — | Shared secret for Next.js on-demand ISR revalidation webhooks. |
| `NEXT_PUBLIC_GA4_MEASUREMENT_ID` | No | — | Google Analytics 4 measurement ID (e.g. `G-XXXXXXXXXX`). Enables client-side analytics when present. |

Minimal `.env.local` for local development:

```bash
MEDUSA_BACKEND_URL=http://localhost:9000
NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY=pk_...   # from Medusa admin → Settings → API Keys
NEXT_PUBLIC_BASE_URL=http://localhost:8000
```

## Running the Dev Server

Start the Medusa backend first, then the storefront:

```bash
# Terminal 1 — in ../turquoise-wholistic
npm run dev

# Terminal 2 — in this repo
yarn dev
```

The storefront will be available at `http://localhost:8000`. The Medusa admin dashboard is at `http://localhost:9000/app`.

`yarn dev` uses Next.js Turbopack (`next dev --turbopack -p 8000`) for fast module reloading. `check-env-variables.js` runs at startup and exits early if `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY` is not set.

## Common Setup Tasks

**Create an admin user** (run in the backend repo `../turquoise-wholistic`):

```bash
npx medusa user -e admin@example.com -p password
```

**Seed demo data** (Canada region, HST tax, 6 wellness products, blog posts, gift cards):

```bash
# In ../turquoise-wholistic
npm run seed
```

**Set up sales channels** to get retail and professional publishable API keys:

```bash
# In ../turquoise-wholistic
npx ts-node src/scripts/setup-sales-channels.ts
```

Copy the printed publishable keys into `.env.local`:

```bash
NEXT_PUBLIC_MEDUSA_RETAIL_PUBLISHABLE_KEY=pk_...
NEXT_PUBLIC_MEDUSA_PROFESSIONAL_PUBLISHABLE_KEY=pk_...
```

**Get a publishable key manually**: Log into the Medusa admin at `http://localhost:9000/app`, go to **Settings → API Keys**, and create or copy a publishable key.

**Configure a region**: The storefront middleware requires at least one region in the Medusa backend. Go to **Settings → Regions** in the admin and add a region (e.g. Canada / CAD) if one does not exist.

## Build and Production

```bash
# Production build (ESLint and TypeScript errors are non-fatal — see next.config.js)
yarn build

# Start the production server on port 8000
yarn start

# Bundle analysis — opens a visual breakdown of the webpack bundles
yarn analyze

# Lint only
yarn lint
```

Note: `ignoreDuringBuilds: true` is set for both ESLint and TypeScript in `next.config.js`. Errors will appear in the output but will not fail the build. Fix them with `yarn lint` and `tsc --noEmit`.

## Troubleshooting

**"No regions found" or middleware redirect loop**

The storefront middleware fetches regions from Medusa on every request. If no regions are configured, it cannot map country codes and may loop. Fix: go to Medusa admin → **Settings → Regions** and create at least one region.

**Channel splash does not appear**

Both `NEXT_PUBLIC_MEDUSA_RETAIL_PUBLISHABLE_KEY` and `NEXT_PUBLIC_MEDUSA_PROFESSIONAL_PUBLISHABLE_KEY` must be set. If either is missing the channel system falls back to `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY` and skips the splash prompt.

**Cart not working or returning 401**

The publishable API key in `.env.local` must match a key associated with the correct sales channel in the Medusa backend. Verify in the admin under **Settings → API Keys** and **Settings → Sales Channels**.

**Build fails immediately with "Missing required environment variables"**

`check-env-variables.js` runs at the start of `next build` and exits if `NEXT_PUBLIC_MEDUSA_PUBLISHABLE_KEY` is not defined. Add it to `.env.local` (or to CI environment secrets).

**Images not loading from S3 or external hosts**

`next.config.js` whitelists specific `remotePatterns` for `next/image`. Add your S3 bucket hostname to the `remotePatterns` array in `next.config.js`:

```js
{
  protocol: "https",
  hostname: "your-bucket.s3.your-region.amazonaws.com",
}
```

Alternatively, set `MEDUSA_CLOUD_S3_HOSTNAME` and `MEDUSA_CLOUD_S3_PATHNAME` environment variables — `next.config.js` reads these and appends a pattern automatically.

**Turnstile CAPTCHA not appearing on the contact form**

`NEXT_PUBLIC_TURNSTILE_SITE_KEY` is not set. Forms degrade gracefully (CAPTCHA widget is hidden, submission still works). To enable it, create a free Cloudflare Turnstile site and add both `NEXT_PUBLIC_TURNSTILE_SITE_KEY` and `TURNSTILE_SECRET_KEY` to `.env.local`.
